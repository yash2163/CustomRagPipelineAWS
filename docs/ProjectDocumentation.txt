1. Project Overview & ScopeThe project aims to build a Metadata-Filtered RAG Agent on AWS. It automates the ingestion of structured recipe data, converts it into a format suitable for Bedrock Knowledge Bases, and provides an intelligent interface for users to query recipes based on both semantic meaning (e.g., "spicy food") and strict constraints (e.g., "cook time < 20 mins").Core Scope:Automated Ingestion: A pipeline that reacts to new S3 uploads and transforms them.Hybrid Search: Custom Lambda logic to combine S3 metadata filtering with vector similarity.Conversational Logic: A Bedrock Agent that manages multi-turn dialogue to collect missing filter parameters.2. Architecture & Service FlowStage A: The Automated Data PipelineLanding Zone (S3 Bucket A): You upload recipes.csv.Trigger: An S3 Event Notification triggers an AWS Lambda (Transformer) or AWS Glue Job.Transformation:Parses CSV.Converts "20 mins" or "1 hr" strings into integer minutes.Generates a pairs of files for each row: recipe_N.txt (content) and recipe_N.txt.metadata.json (metadata).Knowledge Base Source (S3 Bucket B): The transformer saves these files here.Sync: The Bedrock Knowledge Base is triggered (via API call in the transformer) to sync S3 Bucket B with S3 Vector Serverless.Stage B: The Intelligent Agent (Inference)User Input: "I want a chicken recipe that takes less than 30 minutes."Bedrock Agent:Case 1 (Parameters Found): Identifies "chicken" as the query and "30" as total_time_min. Calls the Action Group Lambda.Case 2 (Parameters Missing): If the user says "Give me a quick recipe," the Agent asks: "How many minutes do you define as 'quick' for your filter?"Action Group Lambda:Constructs an S3 VectorsBuckets query: WHERE total_time_min <= 30 AND vector_matches('chicken').Queries S3 VectorsBuckets Serverless.Response Generation: The Lambda returns the recipe chunks to the Agent, which formats the final answer for the user.3.
