Project: Metadata-Filtered Recipe RAGObjective: To build a specialized Retrieval-Augmented Generation (RAG) system that allows users to query structured data using both semantic meaning (natural language) and strict constraints (metadata filters).1. The Strategy: Why this Architecture?Standard RAG systems often struggle with "hard" constraints (e.g., "Find a recipe under 30 minutes"). An LLM might find a recipe that mentions "30 minutes" in the text but actually takes 2 hours to cook.We chose this "Lambda-as-a-Tool" approach because:Precision: It forces the Vector Store to filter by exact numerical values before performing a semantic search.Automation: The pipeline ensures that any new CSV data uploaded is automatically indexed without manual intervention.Cost-Efficiency: By using S3 as a primary store and triggering compute only when needed, we maintain a low-cost, serverless footprint.2. The Technical WorkflowStage A: The Automated Ingestion PipelineLanding Zone: Raw structured data (CSV) is uploaded to a Source S3 Bucket.Transformation (LambdaS3): An S3 Event Trigger fires a Lambda function that parses the CSV. It normalizes data (e.g., converting "1 hr" to 60 minutes) and splits each row into a .txt content file and a .metadata.json sidecar file.Knowledge Base (KB) Sync: These pairs are saved to a Destination S3 Bucket, which serves as the data source for the Amazon Bedrock Knowledge Base.Stage B: The Intelligent Retrieval LayerUser Intent Extraction: The user asks a question via the Bedrock Agent.Parameter Mapping: Using an OpenAPI Schema, the Agent identifies if the user mentioned a constraint (like max_cook_time).The Action Group (BedrockTriggerLambdaQueryKB): Instead of a blind search, the Agent triggers this Lambda, passing the extracted filters as parameters.Metadata-Filtered Query: The Lambda executes a retrieve call to the Knowledge Base, injecting a retrievalConfiguration that limits the search to only those files matching the user's numeric constraints.Stage C: Final ResponseContext Injection: The Lambda returns the filtered, relevant recipe chunks back to the Bedrock Agent.Grounded Answer: Bedrock synthesizes the final response, citing the specific S3 sources for transparency.3.
